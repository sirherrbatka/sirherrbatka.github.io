1.2 DONE Przechowywanie danych   important easy

1.2.1 dla activity

  1. przed przejściem activity do zniszczenia wywoływany jest callback onSaveInstanceState(Bundle bundle)
  2. Bundle pozwala na przechowywanie stanu.

1.3 DONE Activity i fragmenty   important

1.3.1 fragmenty

  1. reprezentują zachowanie części interfejsu w activity
  2. mogą być wykorzystane ponownie w wielu activity
  3. mogą być użyte w większej liczbie w jednym activity
  4. posiada własny cykl życia
  5. musi siedzieć w activity, ponieważ niczym u tasiemca, jego cykl życia jest powiązany z cyklem activity
  6. gdy activity jeż działa (w stanie resumed) można swodobnie dodawać i usuwać fragmenty
  7. tworzenie
    1. subklasa Fragment
    2. kod zbliżony do activity
      1. onCreate
      2. onStart
      3. onPause
      4. onStop
    3. wbudowane fragmenty
      1. DialogFragment
      2. ListFragment
      3. PreferenceFragmentCompat
    4. ładowanie layoutu

      public static class ExampleFragment extends Fragment { @Override public View onCreateView(LayoutInflater inflater, ViewGroup container, Bundle savedInstanceState) { // Inflate the layout for this fragment return inflater.inflate(R.layout.examplefragment, container, false); } }

      1. argumenty inflate
        1. ID
        2. ViewGroup (rodzic)
        3. boolean, informacja o tym czy fragment ma zostać dołączony do grupy w trakcie inflacji
    5. dodawanie do aktywności
      1. dodaj do pliku layoutu
      2. ewentualnie można zrobić to programistycznie
        1. FragmentManager fragmentManager = getSupportFragmentManager(); FragmentTransaction fragmentTransaction = fragmentManager.beginTransaction();
        2. ExampleFragment fragment = new ExampleFragment();

          fragmentTransaction.add(R.id.fragmentcontainer, fragment); //pierwszy argument to ViewGroup do którego ma zostać dodany fragment fragmentTransaction.addToBackStack(fragment); fragmentTransaction.commit();

      3. pobranie activity
        1. metoda call activity
      4. pobranie dodanego fragmentu
        1. ExampleFragment fragment = (ExampleFragment) getFragmentManager().findFragmentById(R.id.examplefragment);

1.5 DONE Manifest i uprawnienia   important easy

1.5.1 plik xml dostarcza informacje o aplikacji dla androida

1.5.2 zawiera

  1. nazwa pakietu javy
  2. opis komponentów aplikacji (activities, services, providers)
  3. nazwy klas implementujące komponenty
  4. procesy hostujące komponenty
  5. deklarację uprawnień
  6. wymagania odnośnie wersji
  7. lista zależności (bibliotek)

1.5.3 uprawnienia

  1. description (ludzki opis)
  2. icon (obrazek)
  3. label (ludzka nazwa)
  4. name (nazwa kodowa)
  5. permissionGroup (przypisanie uprawnienia do grupy)
  6. protectionlevel
    1. normal – izolowany dostęp do funkcji apliakcji. Nie pyta użytkownika o zgodę.
    2. dangerous – dostęp do prywatnych danych. Może prosić o pozwolenie.
    3. signature – zezwala na dostęp aplikacjom podpisanym tym samym certyfikatem.
    4. signatureOrSystem – jak wyżej ale dodatkowo pozwala na dostęp aplikacjom systemowym.

2.3 DONE Podstawowe klasy systemowe (NSString, NSArray, NSDictionary, NSSet, …)   important easy

2.3.1 stringi

  1. NSSstring – niemutowalny string (UTF-16, możliwe utworzenie z UTF lub ascii)
  2. NSMutableString – mutowalny string (UTF-16, możliwe utworzenie z UTF lub ascii)
  3. metody
    1. compare
    2. hash
    3. isEqual
    4. getCString
    5. utf8String
    6. length
    7. write – zapisz string do pliku (wtf?! why?).

2.3.2 array

  1. NSArray – stały, statyczny array
  2. NSMutableArray – dynamiczny array
    1. dodatkowe metody
      1. insert (rozsuń by zrobić miejsce, wsadź do środka)
      2. add (dodaj na koniec)
  3. NSPointerArray – potrafi trzymać nil (yay!)
    1. ma bardziej rozbudowane semantykę do obsługi pamięci (może być zainicjalizowane by trzymać weak pointery)
    2. dodatkowe gówno-metody
      1. insertPointer
      2. addPointer
    3. dodatkowo to co w mutable array
  4. metody
    1. count
    2. contains
    3. firstObject – z początku array
    4. lastObject – na końcu array
    5. subscript – dereferencja elementu z array
    6. index – pozycja elementu w array
    7. indexOfObject – przyjmuje predykat, zwraca indeks pierwszego elementu dla którego predykat jest spełniony.

2.3.3 słowniki

  1. mapuje stringi na obiekty
  2. NSDictionary
  3. NSMutableDictionary
    1. dodatkowe metody
      1. setValue
      2. removeObject
  4. metody
    1. count
    2. allKeys
    3. allValues
    4. value

2.3.4 sety

  1. zbíór (obiekt albo w nim jest, albo go w nim nie ma).
  2. NSSet
  3. NSMutableSet
    1. dodatkowe metody
      1. union
      2. minus
      3. intersect
      4. setSet
    2. add
    3. filter
    4. remove
  4. NSCountedSet
    1. mutowalny set, pozwala na wielokrotne dodanie tego samego elementu
    2. dodatkowe mestody
      1. count
  5. metody
    1. contains
    2. filtered (tworzy set na podstawie istniejącego, zawierający jedynie elementy dla których predykat zwraca true).
    3. member – zwraca obiekt jeśli jest zawarty w zbiorze.
    4. interects
